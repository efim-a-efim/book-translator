<objective>
Swap language order so the TARGET (translated) language comes FIRST and the SOURCE language comes AFTER it, in every output mode that emits paired text.

Purpose: Reader-facing requirement — the translation is the primary content; the original is secondary/reference.
Output: Updated `html_gen.py` emit logic + updated tests proving target-first ordering and interactive default-visible = target.

Scope analysis (already investigated):
- Modes producing paired source/target text live entirely in `src/book_translator/assembler/html_gen.py`:
  - `build_pair_html` → used by per-page (`build`) AND per-sentence bilingual output.
  - `build_interactive_html` → used by interactive (`build_interactive`).
- `EpubBuilder.build_monolingual` emits translation-ONLY (no source interleaving) — already target-only, NOT affected, do not touch.
- Semantics of CSS classes in this codebase: `bt-orig` / `bt-original` = SOURCE (from `para.raw_html` / `para.text`); `bt-trans` / `bt-translation` = TARGET (from `para.translation`). "Swap order" = move the TARGET element before the SOURCE element in pair mode, and move the TARGET into the always-visible `<summary>` in interactive mode.
</objective>
