<objective>
Split the conflated `--mode` flag into two orthogonal flags:
- NEW `--output-mode {parallel|interactive|monolingual}` selects the OUTPUT FORMAT (which assembler runs).
- `--mode {per-page|per-sentence}` retains ONLY GRANULARITY: whether each unit is a paragraph (per-page) or a sentence (per-sentence).

Interaction: when `--output-mode interactive`, `--mode` decides whether each PARAGRAPH (per-page) or each SENTENCE (per-sentence) is packed into a foldable `<details>` block.

Purpose: Clean separation of "what granularity is translated/packed" from "what output format is produced". Today `--mode` mixes both (`VALID_MODES = {per-page, per-sentence, monolingual, interactive}` in src/book_translator/cli.py:28), which makes "interactive at sentence granularity" inexpressible.

Output: Updated CLI parsing/dispatch, sentence-aware interactive HTML, updated tests.
</objective>
