starting around line 101. The current order is:

  1. `if para.kind in ("image", "table"):` — correct, keep first
  2. `elif para.translation:` — WRONG position, catches headings with translations
  3. `elif para.kind == "heading":` — never fires for translated headings

Swap branches 2 and 3 so heading check comes before translation check:

  1. `if para.kind in ("image", "table"):`
  2. `elif para.kind == "heading":` — render as `<h2>` always
  3. `elif para.translation:` — render as `<p>` for non-heading translated text

The heading branch body stays identical: `content_parts.append(f"<h2>{_html.escape(para.text)}</h2>")`.
The translation branch body stays identical: `content_parts.append(f"<p>{_html.escape(para.translation)}</p>")`.
