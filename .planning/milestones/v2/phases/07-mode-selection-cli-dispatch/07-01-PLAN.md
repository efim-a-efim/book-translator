<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> Typer parser | Untrusted user-provided option values enter the command. |
| Parsed options -> run store | Validated command state crosses into persistent run metadata. |
| Mode dispatch -> translation pipeline | User-selected mode determines whether existing per-page translation runs or a future mode is rejected. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | `translate_cmd` mode dispatch | mitigate | Recognized future modes must fail with selected-mode-specific not-yet-implemented errors, never silently falling back to per-page. |
| T-07-02 | Tampering | `JobStore.create_run(...)` boundary | mitigate | Run all mode and cross-option validation before `JobStore()`, `JobMeta(...)`, and `store.create_run(meta)`; assert `tmp_store` remains empty. |
| T-07-03 | Repudiation | CLI error output | mitigate | Errors for invalid flag combinations must name the conflicting option and required mode so users can understand why no run started. |
| T-07-SC | Tampering | Package installs | accept | Phase 7 installs no npm, pip, or cargo packages; use stdlib `enum` and existing Typer dependency only. |
</threat_model>
