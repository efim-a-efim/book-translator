# Plan 05-03: `list` command + `cleanup` subcommand

**Objective:** Add `book-translator list` (D-20) and `book-translator cleanup` (D-19)
as separate Typer commands to `cli.py`. Cover run state filtering, output formatting,
and non-interactive cleanup of terminal-state runs.

**Wave:** 3
**Depends on:** 05-02

**Decision: `--cleanup` is a subcommand `cleanup` (not a root flag).**

Rationale: Typer's idiomatic pattern is sub-commands. A root-level `--cleanup` flag
that triggers a destructive operation is non-standard, ambiguous when combined with
other flags, and inconsistent with Typer's argument model. `book-translator cleanup`
is self-documenting and composable. Behavior locked in D-19 is fully preserved.

```python
@app.command(name="list")
def list_cmd() -> None:
    """List preserved translation runs (failed and unknown state)."""
    store = JobStore()
    runs = store.list_run_metas()
    if not runs:
        typer.echo("No preserved runs found.")
        return
    typer.echo(f"{'RUN ID':<14}  {'DATE':<26}  {'STATE':<10}  PATH")
    typer.echo("-" * 80)
    for run_id, meta in runs:
        state = meta.params.get("state", STATE_UNKNOWN)
        started = meta.params.get("started_at", "unknown")
        run_path = store.run_dir(run_id)
        typer.echo(f"{run_id:<14}  {started:<26}  {state:<10}  {run_path}")
```

**Output format (D-20):** one run per line, columns: run ID (12 chars padded), ISO date
(from `started_at`), state, full path. No colors, no Rich panels (D-08). Header row
with separator. If no runs, print "No preserved runs found." and exit 0.

**Important:** Since successful runs auto-delete (D-18), `list` output will predominantly
show `failed` / `unknown` state runs. `completed` runs appear only in the brief window
between EPUB placement and deletion (effectively never in normal usage).
</action>

```python
@app.command(name="cleanup")
def cleanup_cmd(
    verbose: bool = typer.Option(False, "--verbose", "-v", help="Show each deleted run ID"),
) -> None:
    """Remove preserved terminal runs (failed + completed). Skips running and unknown."""
    store = JobStore()
    runs = store.list_run_metas()
    to_delete = [
        (run_id, meta) for run_id, meta in runs
        if meta.params.get("state", STATE_UNKNOWN) in TERMINAL_STATES
    ]
    if not to_delete:
        typer.echo("Nothing to clean up.")
        return
    removed = []
    errors = []
    for run_id, meta in to_delete:
        try:
            store.delete_run(run_id)
            removed.append(run_id)
            if verbose:
                typer.echo(f"Deleted: {run_id}")
        except Exception as exc:
            errors.append((run_id, exc))
            typer.echo(f"Warning: could not delete {run_id}: {exc}", err=True)
    typer.echo(f"Removed {len(removed)} run(s): {', '.join(removed)}")
    if errors:
        typer.echo(f"{len(errors)} run(s) could not be deleted (see warnings above).", err=True)
```

**Key behaviors (D-19):**
- `TERMINAL_STATES = {"failed", "completed"}` — only these are deleted.
- `STATE_RUNNING` and `STATE_UNKNOWN` are explicitly skipped (filtered out via `in TERMINAL_STATES`).
- Non-interactive: no confirmation prompt.
- Prints count and list of removed run IDs on stdout.
- Individual delete errors are warnings (not fatal); reported on stderr.
- Exit code 0 always (cleanup is best-effort; partial success is not a failure).
</action>

## Threat Model

| Threat | Category | Component | Disposition | Mitigation |
|--------|----------|-----------|-------------|------------|
| T-05-07 | Availability | `cleanup` deletes `running` state run if CLI crashed mid-run leaving stale `running` state | accept | Running-state runs are explicitly skipped. Stale `running` state (from a crashed CLI) appears as `unknown` if meta is corrupt, or stays `running` — neither is in `TERMINAL_STATES`. User can manually delete with `rm -rf`. |
| T-05-08 | Availability | `cleanup` invoked during an active `translate` run (race condition) | accept | `running` state runs are skipped by filter. Race is: translate sets `completed` after delete loop reads the list but before the delete executes — harmless, the run dir would be gone already. |
| T-05-09 | Info Disclosure | `list` displays full `input_path` from `meta.params` | accept | Local CLI; user ran the translation themselves. Path is their own file path. |
