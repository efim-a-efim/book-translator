**Public function:**
```python
def split_chapter_parts(
    pairs: list[str],
    title_html: str,
    chapter_num: int,
    size_limit: int = 300_000,
) -> list[tuple[str, str]]:
```

- `pairs`: list of HTML strings, one per paragraph pair (output of `build_pair_html` calls).
- `title_html`: heading HTML for Part 1 (e.g. `'<h1>Chapter One</h1>'`). Empty string if chapter has no title.
- `chapter_num`: 1-based integer chapter index, used in file names.
- Returns: list of `(body_html, filename)` tuples — one tuple per spine entry.

**Algorithm (greedy fill):**
1. Initialise: `parts = []`, `current_pairs: list[str] = []`, `part_num = 1`.
2. Compute Part 1 base size as `len(title_html.encode("utf-8"))`.
3. For each pair string in `pairs`:
   - Measure `pair_bytes = len(pair.encode("utf-8"))`.
   - If `current_pairs` is non-empty (i.e. at least one pair already buffered) AND adding this pair would exceed `size_limit`:
     - Flush: build `body_html` = `title_html + "".join(current_pairs)` if `part_num == 1` else `"".join(current_pairs)`.
     - Append `(body_html, f"chapter-{chapter_num:02d}-pt{part_num}.xhtml")` to `parts`.
     - Increment `part_num`, reset `current_pairs = [pair]`, reset `current_size = pair_bytes`.
   - Else: append pair to `current_pairs`, accumulate `current_size`.
4. After loop: flush remaining `current_pairs` as the last part (same body_html construction as step 3).
5. Return `parts`.

**Key invariants:**
- A single pair that exceeds `size_limit` alone MUST still be included as its own part (never skip pairs). This is enforced by the "current_pairs is non-empty before flushing" condition in step 3.
- Single-part chapters (no splits needed) still produce `[("...", "chapter-{N:02d}-pt1.xhtml")]` (one entry with `pt1`), satisfying D-12.
- Part 1 body_html = `title_html + joined_pairs` (D-13: title heading in Part 1 only).
- Parts 2+ body_html = `joined_pairs` only (no title, per D-13).

Imports: `from __future__ import annotations`.

**Class:**
```python
class EpubBuilder:
    def build(
        self,
        doc: BookDocument,
        target_lang: str,
        book_id: str = "",
    ) -> epub.EpubBook:
```

**Implementation steps inside `build()`:**

1. **Create and configure book:**
   ```python
   book = epub.EpubBook()
   book.set_identifier(book_id or str(uuid.uuid4()))
   book.set_title(doc.title)          # D-07: copy unchanged
   book.add_author(doc.author)        # D-07: copy unchanged
   book.set_language(target_lang)     # D-08: target lang only
   ```

2. **Add boilerplate items:**
   ```python
   book.add_item(epub.EpubNcx())
   book.add_item(epub.EpubNav())
   ```

3. **Process chapters** (`enumerate(doc.chapters, 1)` → `chapter_num, chapter`):
   - Build title heading HTML: `title_html = f'<h1>{html.escape(chapter.title)}</h1>'` if `chapter.title` else `""`.
   - Build pairs list: `pairs = [build_pair_html(p) for p in chapter.paragraphs]`.
   - Call `split_chapter_parts(pairs, title_html, chapter_num)` → `parts: list[tuple[str, str]]`.
   - For each `(body_html, filename)` in `parts`:
     - `xhtml_content = wrap_chapter_xhtml(body_html, chapter.title)`
     - `ch_item = epub.EpubHtml(title=chapter.title, file_name=filename, lang=target_lang)`
     - `ch_item.content = xhtml_content`
     - `book.add_item(ch_item)`
   - Collect all part items for this chapter into `chapter_items`.

4. **Build TOC and spine** after processing all chapters:
   - Spine: `book.spine = ["nav"] + all_chapter_items` (flat list, one entry per spine item across all chapters).
   - TOC: for each chapter's items list `chapter_items`:
     - If len == 1: add `epub.Link(href=chapter_items[0].file_name, title=chapter.title, uid=chapter_items[0].file_name)` directly to toc list.
     - If len > 1: add `(epub.Section(chapter.title), [epub.Link(href=item.file_name, title=f"{chapter.title} (Part {k})", uid=item.file_name) for k, item in enumerate(chapter_items, 1)])`.
   - `book.toc = tuple(toc_entries)`

5. Return `book`.

## Threat Model

| Threat | Category | Component | Disposition | Mitigation |
|--------|----------|-----------|-------------|------------|
| T-04-04 | DoS | `split_chapter_parts` — pair larger than `size_limit` could cause infinite loop | mitigate | Flush condition requires `current_pairs` non-empty before splitting; guarantees at least one pair per part |
| T-04-05 | DoS | `EpubBuilder.build()` — very large BookDocument held in memory | accept | Chapter-by-chapter processing limits peak memory; prose books are bounded in practice |
| T-04-06 | Injection | `builder.py` — chapter title in `f'<h1>{...}</h1>'` | mitigate | `html.escape()` applied to `chapter.title` before insertion into heading HTML |
