# Phase 10.1: Fix SENT-06: align sentence rendering with chunk-based translations - Context

**Gathered:** 2026-06-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the per-sentence EPUB rendering so each chunk's original text is correctly paired with its translation. The renderer currently re-splits `para.text` using a regex splitter, but `para.sentence_translations` is chunk-level (Punkt+merge rules produce fewer, larger units than raw sentence splitting). This causes count mismatches and wrong text-to-translation pairings.

</domain>

<decisions>
## Implementation Decisions

### Data Model
- Add `sentence_chunk_texts: list[str] | None` to `Paragraph` alongside `sentence_translations`
- Populate in `translate_sentence()` when building `sentence_translations` — store the chunk text for each translation entry
- Field uses Pydantic optional pattern (same as `sentence_translations`) — serialized via Pydantic to_json/from_json automatically, ensuring run-resumption round-trips

### Rendering
- Use `sentence_chunk_texts[i]` as the "original" text per chunk in `build_pair_html()` — this is the Punkt-merged chunk text that was actually translated
- Best-effort on length mismatch: render `min(len(chunk_texts), len(translations))` pairs; skip extras silently
- Preserve existing `bt-pair`/`bt-orig`/`bt-trans` CSS classes — no new CSS needed

### Claude's Discretion
- `_split_sentences_for_rendering()` can be removed or kept as dead code (prefer remove for cleanliness)
- Tests should verify: count match between chunk_texts and translations, rendering output structure, mismatch fallback

</decisions>
