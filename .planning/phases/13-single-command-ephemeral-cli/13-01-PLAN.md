## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → filesystem | User-supplied INPUT path + `--output` path cross into file I/O (copy, parse, atomic move) |
| Process → OS temp filesystem | mkdtemp run dir created and recursively deleted under `$TMPDIR` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Information Disclosure | Predictable run-dir path / world-readable temp | mitigate | `tempfile.mkdtemp(prefix="book-translator-")` yields an unpredictable name with 0700 (owner-only) perms; do NOT loosen (Pitfall 2). Path printed only under debug-posture flags (D-05). |
| T-13-02 | Tampering / Denial | `shutil.rmtree` following a symlink or hitting a locked path | mitigate | `shutil.rmtree` does not follow top-level symlinks by default (no `os.path.islink`-cwd traversal); run dir is created by mkdtemp under owner-only perms so an attacker cannot pre-seed a symlink inside it. rmtree wrapped in `try/except OSError`; on a successful run a cleanup failure warns and exits 0 (D-09), never destroying the already-written output (Pitfall 3 — output moved before cleanup). |
| T-13-03 | Denial of Service | Crash / Ctrl-C leaving temp state on disk | mitigate | `try/finally` guarantees rmtree runs on exceptions and `KeyboardInterrupt` (D-08); no on-disk state survives a clean default run. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | This phase installs NO external packages (RESEARCH Package Legitimacy Audit: N/A). Net dependency change is removal of two internal modules. No legitimacy checkpoint required. |

No HIGH-severity threats identified — local CLI refactor, no network/auth/persistence additions. Block-on-high gate has nothing to block.
