# Plan 02-03: TXT + Markdown Parsers + Tests

**Phase:** 2 — Parsers
**Wave:** 3
**Status:** Not Started

## Objective
Implement `TxtParser` and `MarkdownParser`, then cover all three parsers (EPUB, TXT, Markdown) with comprehensive tests in `tests/test_parsers.py`.

## Requirements
REQ-2, REQ-3

## Context
Per D-07–D-09, TXT files split into chapters on HR rulers and into paragraphs on blank lines; single newlines within a block are continuation lines. Per D-10–D-12, the Markdown parser converts MD → HTML via the `markdown` library (with `tables` extension) then reuses `_extract_blocks` imported from `epub.py`. Tests use in-memory EPUB construction (`epub.EpubBook()` + `BytesIO`) for valid-EPUB fixtures and `zipfile` stdlib for DRM/traversal fixtures.

## Tasks

### Task 02-03.1: `TxtParser`
**Type:** implementation
**Files modified:**
- `src/book_translator/parsers/txt.py` — new file; `TxtParser` class

**Steps:**
1. Create `src/book_translator/parsers/txt.py` with `from __future__ import annotations` header.
2. Imports: `re`, `pathlib.Path`, `book_translator.parsers.ParseError`, `book_translator.models.document` (Paragraph, Chapter, BookDocument).
3. Define module-level regex constants:
   - `_RULER_RE = re.compile(r"^\s*[-*_]{3,}\s*$", re.MULTILINE)` — matches HR rulers per D-07
   - `_BLANK_RE = re.compile(r"\n{2,}")` — splits on one or more blank lines per D-09
4. Implement `_read_text(path: Path) -> str`: try `path.read_text(encoding="utf-8")`; on `UnicodeDecodeError`, fall back to `path.read_text(encoding="latin-1")`. (Agent's Discretion: UTF-8 first, latin-1 fallback.)
5. Implement `class TxtParser:` with `def parse(self, path: Path) -> BookDocument:`:
   - Read file with `_read_text(path)`.
   - Split text on `_RULER_RE`: `sections = _RULER_RE.split(text)`.
   - `is_multi = len(sections) > 1`. Per D-08: chapter title is `""` for multi-chapter, `path.stem` for single-chapter.
   - Maintain `chapter_idx = 0` (increments only for non-empty chapters).
   - For each raw section: strip it; skip if empty.
   - `chapter_id = f"ch_{chapter_idx:03d}"`, `title = "" if is_multi else path.stem`.
   - Split section on `_BLANK_RE` for raw paragraph blocks. For each block: strip it; skip if empty. Join internal single newlines with a space: `block.replace("\n", " ")`. Use a separate `para_counter = 0` (increments only for non-empty paragraphs). Append `Paragraph(id=f"{chapter_id}:{para_counter}", text=block, raw_html=f"<p>{block}</p>", kind="paragraph")`.
   - Skip chapter if it produces no paragraphs.
   - Append `Chapter(id=chapter_id, title=title, paragraphs=paras)`. Increment `chapter_idx`.
   - Return `BookDocument(title=path.stem, chapters=chapters)`.

**Verification:** `python -c "from book_translator.parsers.txt import TxtParser; print('OK')"` exits 0.

### Task 02-03.2: `MarkdownParser`
**Type:** implementation
**Files modified:**
- `src/book_translator/parsers/md.py` — new file; `MarkdownParser` class

**Steps:**
1. Create `src/book_translator/parsers/md.py` with `from __future__ import annotations` header.
2. Imports: `markdown`, `pathlib.Path`, `bs4.BeautifulSoup`, `book_translator.parsers.epub._extract_blocks`, `book_translator.parsers.ParseError`, `book_translator.models.document` (Chapter, BookDocument).
3. Encoding helper (inline — do not import from `txt.py`):
   ```
   def _read_text(path: Path) -> str:
       try: return path.read_text(encoding="utf-8")
       except UnicodeDecodeError: return path.read_text(encoding="latin-1")
   ```
4. Implement `class MarkdownParser:` with `def parse(self, path: Path) -> BookDocument:`:
   - Read file with `_read_text(path)`.
   - Convert: `html = markdown.markdown(text, extensions=["tables"])`. The `tables` extension is required to produce `<table>` elements from pipe tables (verified in RESEARCH.md).
   - Parse: `soup = BeautifulSoup(html, "lxml")`. Extract top-level body children: `body = soup.find("body") or soup`; `top_level = [c for c in body.children if hasattr(c, "name") and c.name]`.
   - **No `<h1>` found (D-12):** `paragraphs = _extract_blocks(html, "ch_000")`; return `BookDocument(title=path.stem, chapters=[Chapter(id="ch_000", title=path.stem, paragraphs=paragraphs)])`.
   - **`<h1>` found (D-11):** Walk `top_level` maintaining state: `current_title = ""`, `current_elements: list[str] = []`, `chapter_idx = 0`, `chapters: list[Chapter] = []`.
     - Define a `flush()` helper: if `current_elements` is non-empty, call `_extract_blocks("".join(current_elements), f"ch_{chapter_idx:03d}")`, append `Chapter(id=f"ch_{chapter_idx:03d}", title=current_title, paragraphs=paragraphs)` (skip if paragraphs is empty), increment `chapter_idx`, reset `current_elements = []`.
     - For each element in `top_level`: if `element.name == "h1"` → call `flush()`; set `current_title = element.get_text(strip=True)`. Else → `current_elements.append(str(element))`.
     - After the loop: call `flush()` one final time for the last chapter.
   - Content before the first `<h1>` (if any) is flushed as an untitled chapter (`title = ""`); this is an edge case but handled naturally by the algorithm.
   - Return `BookDocument(title=path.stem, chapters=chapters)`.
5. `__all__ = ["MarkdownParser"]` at module bottom.

**Verification:** `python -c "from book_translator.parsers.md import MarkdownParser; print('OK')"` exits 0.

### Task 02-03.3: EPUB parser tests
**Type:** test
**Files modified:**
- `tests/test_parsers.py` — new file; EPUB test functions

**Steps:**
1. Create `tests/test_parsers.py`. Imports: `io.BytesIO`, `zipfile`, `ebooklib.epub`, `pytest`, `pathlib.Path`, `book_translator.parsers.epub.EpubParser`, `book_translator.parsers.ParseError`.

2. Add `_make_epub(chapters: list[tuple[str, str, str]]) -> bytes` helper:
   - Signature: `(item_id, title, html_body)` per chapter.
   - Create `epub.EpubBook()`, set `book.set_title("Test")`.
   - For each chapter tuple: create `item = epub.EpubHtml(uid=item_id, file_name=f"{item_id}.xhtml")`; `item.content = f"<html><body>{html_body}</body></html>".encode()`; `book.add_item(item)`.
   - Set `book.spine = [(item_id, "yes") for item_id, _, _ in chapters]`.
   - Set `book.toc = ()`.
   - `buf = BytesIO()`; `epub.write_epub(buf, book)`; `buf.seek(0)`; return `buf.read()`.

3. Add `_make_drm_epub() -> bytes` helper:
   - `buf = BytesIO()`; open `zipfile.ZipFile(buf, "w")`; write `mimetype` entry (value `"application/epub+zip"`, no compression); write `META-INF/encryption.xml` (empty bytes); return `buf.getvalue()`.

4. Add `_make_traversal_epub() -> bytes` helper:
   - Same as above but write `"../evil.txt"` instead of `encryption.xml`.

5. Test functions (each a standalone `def test_...` using `tmp_path` fixture):
   - `test_simple_epub`: `<p>Hello world</p>` → 1 chapter, 1 paragraph, `text="Hello world"`, `kind="paragraph"`, `id` matches `"<item_id>:0"`, `raw_html` contains `"<p>"`.
   - `test_heading_kind`: `<h2>Sub</h2><p>Body</p>` → first paragraph `kind="heading"`, `text="Sub"`; second `kind="paragraph"`.
   - `test_blockquote_is_caption`: `<blockquote>Quote here</blockquote>` → `kind="caption"`.
   - `test_image_empty_text`: `<img src="img.jpg" alt="cover"/>` → `kind="image"`, `text=""` (D-06).
   - `test_table_kind`: `<table><tr><td>data</td></tr></table>` → `kind="table"`.
   - `test_empty_paragraph_skipped`: `<p></p><p>Real</p>` → exactly 1 paragraph extracted (D-04).
   - `test_nested_div_no_double_extract`: `<div><p>Inner</p></div>` → exactly 1 paragraph (not 2), `text="Inner"`.
   - `test_blockquote_is_leaf_not_descended`: `<blockquote><p>Nested</p></blockquote>` → exactly 1 paragraph with `kind="caption"` (blockquote not descended into; `<p>` inside it is not extracted separately).
   - `test_nav_item_excluded`: Build EPUB with `epub.EpubNav()` added to book; verify nav item ID does not appear in any chapter of the parsed `BookDocument`.
   - `test_drm_raises`: Write `_make_drm_epub()` bytes to `tmp_path / "drm.epub"`; assert `EpubParser().parse(...)` raises `ParseError` and the message contains `"DRM"`.
   - `test_zip_traversal_raises`: Write `_make_traversal_epub()` to `tmp_path / "traversal.epub"`; assert raises `ParseError` with `"Unsafe"` in message.

**Verification:** `python -m pytest tests/test_parsers.py -k "epub or drm or traversal or nav" -v` — all EPUB-related tests pass.

## Threat Model
- **Arbitrary path access (T-03-01):** Both `TxtParser` and `MarkdownParser` call `Path.read_text()` on caller-supplied paths. Path validation is the CLI layer's responsibility (Phase 5). Disposition: accept.
- **Malicious Markdown input (T-03-02):** The `markdown` library produces HTML; BeautifulSoup with `lxml` parses it. No `eval`/`exec` paths exist. Disposition: accept.
- **Large file memory exhaustion (T-03-03):** Entire file is read into memory. Local CLI tool — no server context. Disposition: accept.

## Wave Dependency Notes
Wave 3: `md.py` imports `_extract_blocks` from `epub.py` (plan 02-02), so this plan is strictly ordered after 02-02. The `TxtParser` itself only depends on 02-01, but is grouped here to keep implementation and tests together in a single execution pass.
