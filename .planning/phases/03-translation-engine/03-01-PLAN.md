<read_first>

- src/book_translator/parsers/txt.py — pure list-slicing / iteration pattern to replicate; no classes, no I/O

<task>
Create `src/book_translator/translator/chunker.py`.

The module exposes one function:

```
def build_context_window(
    flat: list[Paragraph],
    idx: int,
    window: int,
) -> tuple[list[Paragraph], list[Paragraph]]:
```

`flat` is the complete flattened paragraph list across ALL chapters (cross-chapter is natural because the caller flattens `[p for ch in doc.chapters for p in ch.paragraphs]` before passing it).
`idx` is the index of the target paragraph in `flat`.
`window` is the number of context paragraphs on each side (D-01: default 3, cross-chapter by design).

Implementation:
- `before = flat[max(0, idx - window):idx]`
- `after = flat[idx + 1 : idx + 1 + window]`
- Return `(before, after)`

No filtering of `kind=image/table` here — the chunker returns ALL surrounding paragraphs regardless of kind. The engine decides whether to include their text in the prompt (see AI-SPEC §4 for the "[non-text element]" placeholder strategy — agent's discretion whether to include this in v1; for now simply include all paragraphs).

The file must begin with `from __future__ import annotations`. Only import `from book_translator.models.document import Paragraph`. No class, no state, no I/O.
</task>

<task>
**Part A — Create `src/book_translator/translator/prompt.py`:**

Two module-level functions, no class:

`build_system_prompt(source_lang: str, target_lang: str) -> str`
Implements D-03: system prompt contains (1) professional literary translator role, (2) language pair `{source_lang}` to `{target_lang}`, (3) instruction to preserve narrative voice, character names, and tone, (4) "Output only the translated text — no explanations, no commentary." Note: `source_lang` and `target_lang` are interpolated ONLY here in the system prompt — they NEVER appear in the user message. This is the injection-safe design.

`build_user_message(paragraph: Paragraph, before: list[Paragraph], after: list[Paragraph]) -> str`
Implements D-04 and D-05:
- For each paragraph in `before`: append `[context] {para.text}` as a line
- Append `<source_text>{paragraph.text}</source_text>` — XML delimiter wraps ONLY the target; context is plain labeled text
- For each paragraph in `after`: append `[context] {para.text}` as a line
- Append `Translate the text inside <source_text> tags.` as the final instruction line
- Return `"\n".join(parts)`

D-05 enforcement: `source_lang` and `target_lang` are NOT interpolated into the user message. The target paragraph text is the raw `paragraph.text` inserted verbatim inside the XML tags — no sanitisation of the text itself (the XML delimiters provide the semantic boundary; the instruction cannot leak out of the XML envelope because the model is told to translate ONLY the content inside those tags).

_make_rate_limit_error() -> RateLimitError
    req = httpx.Request("POST", "https://api.openai.com/v1/chat/completions")
    resp = httpx.Response(status_code=429, request=req)
    return RateLimitError("Rate limited", response=resp, body=None)

_make_server_error(status=503) -> APIStatusError
    req = httpx.Request("POST", "https://api.openai.com/v1/chat/completions")
    resp = httpx.Response(status_code=status, request=req)

    req = httpx.Request("POST", "https://api.openai.com/v1/chat/completions")
