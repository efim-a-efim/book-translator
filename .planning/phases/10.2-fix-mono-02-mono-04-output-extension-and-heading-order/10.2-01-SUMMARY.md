# Phase 10.2 Plan 01: Fix MONO-02 + MONO-04 Output Extension and Heading Order Summary

Fixed two silent correctness bugs in monolingual mode: MONO-04 (headings with translations rendered as `<p>` instead of `<h2>`) and MONO-02 (default output path always used `.epub` regardless of `--output-format`).

## What Was Done

### MONO-04 (builder.py)

In `build_monolingual()`, the inner paragraph loop had wrong `elif` ordering:
- Old: `image/table` → `elif para.translation` → `elif para.kind == "heading"`
- Fixed: `image/table` → `elif para.kind == "heading"` → `elif para.translation`

A heading paragraph with a translation was previously caught by the `elif para.translation` branch, rendering as `<p>`. After the fix, heading paragraphs always render as `<h2>`.

### MONO-02 (cli.py)

Added `FORMAT_TO_EXT: dict[str, str] = {"epub": ".epub", "txt": ".txt", "md": ".md"}` after `VALID_OUTPUT_FORMATS`.

Replaced hardcoded `f"{stem}.{target_lang}.epub"` with conditional extension derivation:
- When mode is monolingual: `_ext = FORMAT_TO_EXT.get(output_format or "epub", ".epub")`
- Otherwise: `_ext = ".epub"`
