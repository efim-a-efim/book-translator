## Task 1: Extend `JobMeta` — no model change (D-21: params dict approach)

**Decision: Store state in `meta.params["state"]` and timestamps in `meta.params`.**

Rationale: `JobMeta` is a plain dataclass with `params: dict`. `JobStore._write_meta`
already serialises the full `params` dict. Using params avoids a breaking model change,
requires no migration shim, and keeps the model generic. Typed state constants are defined
as module-level literals in `job_store.py` so callers never use bare strings.

States: `"running"` | `"failed"` | `"completed"` | `"unknown"`.
Timestamps: `meta.params["started_at"]` and `meta.params["finished_at"]` (ISO 8601 UTC).
Input metadata: `meta.params["input_filename"]`, `meta.params["input_path"]`.

**`delete_run(self, run_id: str) -> None`:**
```python
def delete_run(self, run_id: str) -> None:
    """Remove a run directory and all its contents."""
    import shutil
    shutil.rmtree(self.run_dir(run_id))
```

**`list_run_metas(self) -> list[tuple[str, JobMeta]]`:**
```python
def list_run_metas(self) -> list[tuple[str, JobMeta]]:
    """Return (run_id, JobMeta) pairs for all existing runs, sorted by run_id."""
    result = []
    for run_id in self.list_runs():
        try:
            meta = self.read_meta(run_id)
        except Exception:
            meta = JobMeta(model="unknown", params={"state": "unknown"})
        result.append((run_id, meta))
    return result
```

Also add `import shutil` at the top of the file (after existing imports).

## Threat Model

| Threat | Category | Component | Disposition | Mitigation |
|--------|----------|-----------|-------------|------------|
